# SunTravel — сервис бронирования отелей

Полноценный travel / hotel booking сервис: каталог из БД → карточка отеля →
выбор дат → проверка доступности → расчёт цены → бронирование → оплата →
подтверждение → «Мои бронирования». Плюс AI-чатбот, связанный с данными сайта,
и адаптивная мобильная вёрстка.

## Стек
- Node.js + Express
- SQLite (sqlite3)
- JWT-авторизация (jsonwebtoken), пароли — bcryptjs
- Чистый фронтенд (HTML/CSS/JS), общий слой `app.js` + `app.css`

## База данных — ровно 3 таблицы
- `users` — пользователи
- `hotels` — отели (заполняются сидом при первом старте)
- `bookings` — бронирования (вся booking-логика живёт здесь)

Никаких дополнительных таблиц. Расчёт цены, статусы оплаты и проверка
доступности реализованы через backend-логику поверх этих трёх таблиц.

## Запуск
```bash
npm install      # установить зависимости (нужен доступ к сети)
npm start        # запустить сервер
```
Откройте http://localhost:3000

> В среде сборки сеть была отключена, поэтому `npm install` здесь не выполнялся.
> Бизнес-логика вынесена в чистые модули `lib/` и покрыта тестами, которые
> запускаются без зависимостей.

## Структура ключевых файлов
- `server.js` — API и статика (источник истины по ценам и доступности)
- `lib/pricing.js` — парсинг/валидация дат, расчёт стоимости, проверка доступности
- `lib/chatbot.js` — логика AI-чатбота, связанная с данными сайта
- `app.js` / `app.css` — общий слой: состояние авторизации в навигации, чатбот
  на каждой странице, глобальная адаптивность
- `landing.js` — оживляет статические страницы-направления (подставляет реальные
  отели из БД, убирает тупиковые переходы)
- `hotels.html` — динамический каталог с фильтрами и сортировкой
- `search-results.html` — живой поиск по `/api/hotels` (фильтры цены/рейтинга,
  сортировка, переход в карточку отеля по slug)
- `hotel-detail.html` — карточка отеля, выбор дат/гостей, проверка доступности
- `payment.html` — оплата конкретной брони (mock-эквайринг)
- `confirmation.html` — подтверждение и «Мои бронирования»

## Что было доработано в этой версии
- `search-results.html` переписан со статики на живой поиск из БД (раньше карточки
  и цены были захардкожены, а все ссылки вели на отель по умолчанию).
- На главной (`index.html`) строка поиска и кнопка «Забронировать» теперь ведут
  в живой поиск; добавлен блок «Популярные отели» из БД с корректными ссылками.
- `destinations.html`: карточки направлений ведут в каталог с фильтром, а
  статистика и превью отелей подтягиваются из БД.
- Страницы-направления (`PhuQuoc.html`, `Thailand.html`) показывают реальные отели
  направления и больше не содержат тупиковых кнопок.
- Глобальная адаптивность: двухколоночные раскладки каталога/поиска корректно
  сворачиваются на телефоне; кнопки авторизации (`.login-btn`) работают на всех страницах.
- AI-чатбот: исправлено распознавание «у моря/острова» (склонения).

## Основные API
- `POST /api/register`, `POST /api/login`, `GET /api/profile`
- `GET /api/hotels` (фильтры: destination, q, min_price, max_price, sort)
- `GET /api/hotels/:slug`
- `POST /api/bookings/quote` — расчёт цены + проверка доступности (без сохранения)
- `POST /api/bookings` — создание брони (сервер сам пересчитывает цену)
- `GET /api/bookings/my`, `GET /api/bookings/:id`
- `POST /api/bookings/:id/pay`, `POST /api/bookings/:id/cancel`
- `POST /api/chat` — AI-чатбот с контекстом страницы и данными пользователя
